# 更新日志

## 2026-06-18

### Web 导入与建库体验

- 重构设置页的数据导入面板，支持上传一个或多个 WeFlow JSON 文件，并按文件展示导入状态、文件变化状态、最近导入时间、消息总数、入库数量和变化数量。
- 新增完整的 Web 导入任务流：支持全流程导入、增量导入、重建全部、仅 FTS、仅分块、仅摘要和仅向量等模式；批量导入会逐个文件执行，并在后端运行冲突时自动停止后续批次。
- 导入任务现在支持取消、任务列表、任务分页、任务互斥、轻量任务状态恢复、进度条、阶段信息、ETA、日志尾部和 WebSocket 实时进度；WebSocket 不可用时前端会回退到轮询。
- 取消导入时，如果任务尚未真正启动或子进程已经被成功终止，后端会立即把任务落为已取消；前端会先合并取消接口返回再静默刷新列表，避免真实使用中出现长时间卡在“取消中”或整块列表闪烁。
- 已取消但后台线程稍后才开始执行的导入任务会在抢占单进程锁前直接退出，避免用户取消后立刻重试时旧任务短暂占锁或把取消状态改写成失败。
- 取消后后台线程释放单进程锁的短暂收尾窗口会返回 409 提示稍后再试，不会创建一个随即失败的新导入任务，避免用户快速重试时任务列表出现误导性错误记录。
- 如果导入子进程已经自然退出但后台线程尚未回写状态，取消接口会按真实退出码落为完成或失败，而不是把已完成的导入误标为已取消。
- 已完成、已取消或失败的导入任务不再根据旧进度事件显示 ETA，避免终态任务仍像在倒计时。
- 导入任务列表和 WebSocket 进度只读取最近日志片段来推断旧式文本进度，避免长时间建库产生的大日志拖慢任务面板刷新。
- 文件列表刷新会按页批量读取文件来源映射、关联会话块和索引缺口统计，避免已导入 JSON 较多时每一行重复触发多次数据库查询。
- 导入任务时间戳提升到毫秒级，并在同一 JSON 快速重试时用稳定排序选择最近任务，避免文件行展示旧任务状态。
- 导入任务列表完整刷新期间会阻止新的“加载更多任务”请求，避免慢网络或快速点击时用旧 offset 合并分页结果。
- 导入任务静默刷新会按后端分页上限补齐已加载页，并只继承同 ID 任务的本地进度字段，避免后台刷新把用户已展开的历史任务列表压回第一页，或继续展示服务端已移除的旧任务。
- 导入面板已有文件或任务数据时，刷新失败会保留当前列表并显示持续错误提示，避免长时间导入过程中因短暂网络错误清空视图。
- 后端上传接口增加安全校验：限制上传大小、拒绝空文件/损坏 JSON/非 WeFlow JSON，保留原始显示文件名，生成安全上传 ID，并阻止路径穿越和非 JSON 文件被导入。上传内容会先写入不可导入的临时文件，校验和元数据写入成功后再原子替换为最终 `.json`，避免上传中断或刷新列表时暴露半截 JSON。
- 文件级操作更贴近真实使用：已变化文件默认使用增量导入，并只开放可用模式；正在运行或取消中的文件会禁用单文件和批量启动入口，避免重复触发任务。
- 导入面板在文件列表刷新、健康诊断刷新和真正启动任务前都会重新校验当前选择的模式；如果摘要/向量能力或索引状态变化导致旧选择不再可用，会自动回落到当前文件允许的模式，避免把已隐藏的模式提交给后端。
- 前端启动锁会同时参考任务列表和文件列表的运行态；即使任务列表刷新失败，只要文件行仍显示导入中，其他文件的启动入口也会保持禁用，避免用户反复点出 409 冲突。
- 上传队列进行中也会锁定单文件和批量导入启动入口，避免大 JSON 上传尚未刷新到文件列表时用户抢跑启动旧列表任务。
- 上传校验改为按探测到的 JSON 编码从临时文件文本流解析，继续兼容 UTF-8 BOM / UTF-16 导出，同时避免为大 JSON 额外复制整份 `bytes` 内容。
- 旧库缺少文件来源映射时，前端会把已导入但无法定位索引范围的 JSON 作为待处理项，只开放增量、全流程或强制重建；后端也会拒绝这类文件的单项 FTS/分块/摘要/向量构建，避免单项修复静默无效。
- 仅摘要和仅向量构建现在要求目标 JSON 已有会话分块；前端会在无会话块时隐藏摘要/向量模式并默认选择仅分块，后端也会在直接 API 调用时提前拒绝，避免任务启动后才失败。
- 导入面板会把已导入但会话块为 0、缺摘要或缺向量的 JSON 纳入批量待处理范围，让“已入库但索引不完整”的文件更容易被继续修复。
- 文件列表的缺摘要/缺向量计数会按当前运行环境过滤：未配置 `SUMMARY_MODEL` 时不把缺摘要当成可自动修复缺口，embedding 或 `sqlite-vec` 不可用时不把缺向量放入待处理范围，避免批量导入反复执行不会产生修复效果的任务。
- “仅摘要”和“仅向量”启动前会校验对应模型配置和向量扩展是否可用；目录目标和单 JSON 目标都会提前返回可操作错误，而不是创建一个随后失败的后台任务。
- 目录目标的单项构建会先确认目录内 JSON 已有可定位的入库消息范围；仅摘要和仅向量还要求已有会话分块，避免未导入目录或无索引范围目录静默完成 0 条修复。
- 导入子进程新增结构化进度事件，普通 CLI 仍保持文本输出；后端只把结构化事件用于任务状态，不把内部事件混进用户可见日志。
- 导入任务失败时会优先把结构化失败进度或最近日志尾部提炼成脱敏错误原因，避免前端只显示 `ingest exited with code 1` 这类无法操作的退出码。
- 旧 `file_path` 目录目标会在创建任务前检查目录内是否存在可导入 `.json` 文件；空目录会直接返回 400，避免任务先启动再失败。
- `/api/ingest/start` 会先修剪 `upload_id`、`file_id` 和 `file_path` 的首尾空白，并把纯空白当作未提供目标；目标字段增加 2048 字符上限，避免异常客户端把超长路径输入压到文件系统解析层。
- 导入任务状态和 WebSocket 进度出口会再次脱敏 `error` 字段，即使未来异常路径写入了原始错误，也不会把 API Key 或 token 透传给前端。
- 后端关闭时会清理运行中和排队的导入任务，取消子进程并更新任务状态，避免重启后留下误导性运行状态。

### WeFlow 解析与摄入流水线

- `core.ingest` 支持递归扫描目录内 JSON，并按稳定路径顺序处理；单个 WeFlow 文件解析失败时可继续处理其他文件，全部不可用时返回清晰错误。
- 增加 `--skip-import` 及单项索引模式语义，用于只基于已有数据库补建 FTS、分块、摘要或向量，避免用户选择单项构建时误导入尚未入库的 JSON。
- `--skip-import` 单项索引构建现在要求目标路径至少匹配一个 JSON，并且这些目标已有可定位的入库消息或会话块范围；空目录、错误路径或从未入库的 JSON 会直接失败，避免无目标范围时误触发全库索引重建或静默完成 0 条修复。
- 显式单项构建现在严格只执行用户选择的阶段：仅 FTS / 仅分块不会因为缺摘要或缺向量而调用模型，仅向量不会为了缺摘要额外调用摘要模型；普通增量、全流程和强制重建仍保留自愈补齐逻辑。
- 单个 JSON 触发缺失索引自愈时，缺失 `seq` 的检测和重算也收敛到目标 JSON 关联会话，避免单文件修复意外扫描全库。
- 文件来源映射现在只记录数据库中真实存在的消息 ID，避免旧脚本或异常重导留下悬空映射后误导文件级索引状态。
- 摘要构建现在可以只依赖显式 `SUMMARY_MODEL`，不再错误要求默认 `CHAT_MODEL` 必须存在；适合先建库、后补摘要的分阶段配置。
- 摘要和 embedding 失败信息会脱敏并压缩，配置缺失时给出可执行提示；向量维度仍可按模型实际返回自动修正。
- 全局摘要自愈现在会把空白摘要与 `NULL` 一样视为缺摘要，分块重建不会复用空白旧摘要，语义检索结果也不会把空白摘要原样暴露给 Agent，避免旧库或异常模型返回空串后普通导入误判为摘要已完成。
- JSON 已成功解析入库后，即使后续摘要或 embedding 阶段因模型/API 错误停止，也会先保存该文件的导入记录，前端可继续显示已入库状态和缺摘要/缺向量数量，方便直接执行单项修复。
- 单 JSON 摘要或向量修复在判断已有向量时会按目标会话 ID 分批查询，不再扫描全库向量表，降低大库环境下单文件修复的数据库压力。
- WeFlow 解析器兼容更多真实导出字段别名，包括消息类型、时间、正文、平台 ID、本地 ID、回复 ID、引用正文和引用发送人等常见变体。
- WeFlow 解析器兼容顶层 `messageList/message_list/msgList/chatRecords/records` 消息列表别名；上传校验和 CLI 导入都会接受这些真实导出结构，避免只因顶层列表字段不叫 `messages` 而拒绝整份 JSON。
- WeFlow 解析器现在兼容数字 `type/msgType/messageType` 消息码，以及 `text/image/voice/video/location/link/file/system` 等短类型别名，减少真实导出里只给数字类型时的消息漏导。
- WeFlow 解析规则升级到可读取 JSON 自带的图片 OCR、语音转写和视频说明字段；没有明确可读文本的媒体消息仍会跳过，避免把原始媒体 payload 混入检索内容。
- 解析规则升级到兼容 `typeName/msgTypeName`、`formattedTime/msgCreateTime`、`msgContent/plainText`、`newMsgId/clientMsgId/localMsgId` 以及 `quote/refer/reply` 嵌套引用对象，减少不同导出版本下的有效聊天漏导。
- 字段别名解析现在会跳过空字符串、`null` 和无效时间主字段，继续读取后续有效别名；修复真实导出中 `type: null`、`createTime: ""`、`quotedContent: ""` 等占位字段导致消息被误跳过的问题。
- 嵌套引用对象也会跳过空的 `quote/refer/reply` 占位对象，并从同一个有效引用对象读取发送人和正文，避免引用正文漏导或发送人错配。
- 引用消息会从嵌套的 `quote/refer/reply` 对象读取被引用消息 ID，并映射到当前 JSON 的 scoped message id，提升后续上下文追溯的准确性。
- 入库时会把部分导出中仍是 raw ID 或 basename scoped ID 的 `reply_to` 规范到同文件作用域下已存在的消息 ID，避免只导入新增回复片段时上下文追溯断链。
- 早期已按随机上传 UUID 路径入库的聊天记录，在同一聊天重新上传后会按稳定上传 scope、raw 消息 ID 后缀和消息字段安全升级，避免旧上传前缀与新稳定前缀并存造成重复消息；部分重导的回复也能指向旧上传父消息。
- 分块逻辑在切分前稳定排序消息，尾部小块会在时间距离允许时合并回前一块，减少过碎会话块影响摘要和检索效果。
- 多行聊天消息写入会话块时会给每一行重复发送人前缀，避免摘要或 embedding 阶段把续行误判成无人称文本或其他人的话。
- 跨日但连续的聊天分块标题会保留结束日期，避免检索证据显示成同一天的时间范围。
- 按线程重算消息序号、读取待重建消息和替换会话分块时会对大量线程列表分批执行，避免大库增量导入或单文件重建涉及很多会话时撞到 SQLite 参数上限。

### 检索、Agent 与工具调用稳定性

- 检索存储层增加大量 direct-call 兜底：非法 `limit`、`offset`、`before`、`after`、时间过滤和空查询不会直接抛异常，而是回退、夹紧或返回结构化空结果。
- 语义检索在没有 query、没有向量、旧库 JSON 字段损坏、过滤条件异常等情况下保持可预期返回，不再整次查询失败。
- 语义检索存储层会自行裁剪超长 query，避免内部 direct-call 绕过工具 schema 后把无界输入发送到 embedding 服务。
- 语义检索现在只在过滤条件规范化后确实有效时才启用“过滤范围内最近会话”兜底；空白会话名或 `not-a-date` 这类无效日期字符串不会误触发最近会话兜底，避免把无关聊天当作答案依据。
- 语义检索的向量召回现在会把会话名和时间过滤下推到 `sqlite-vec` 候选范围内，避免大库中目标群聊/日期的相关会话块被全库 top-k 结果挤掉后误判无命中。
- 语义检索最终过滤的会话名匹配现在与 SQLite `LIKE` 一样大小写不敏感，避免英文群名大小写不同导致已命中的会话块被误判为只能返回最近上下文。
- 关键词检索对短词使用 `LIKE` 兜底，并限制过多查询词，避免异常输入导致 SQL 过重或返回数据库内部细节。
- 关键词检索命中长消息时，预览会围绕第一个命中词截取，避免结果列表显示“命中了但看不到命中内容”。
- Agent 工具参数改由 Pydantic schema 提前限制窗口大小：关键词检索、语义检索、上下文追溯和时间浏览都会在进入数据库前拒绝过大的请求。
- 关键词检索和语义检索工具会把空白的可选 `after`/`before` 视为未设置，避免模型生成空字符串时间过滤时直接校验失败。
- Agent 直接调用会先规范化问题，空问题或非字符串问题不会初始化模型；历史裁剪会移除过期工具消息和非法开头上下文，避免发送给模型的消息序列不合法。
- Agent 每轮会把当前启用/停用工具策略追加到系统提示词；当用户通过设置关闭 `semantic_search` 等工具时，即使自定义提示词仍保留旧路由规则，模型也不会继续被引导去调用已停用工具。
- 普通和流式 Agent 都能处理畸形 tool call：错误会转成可读工具错误继续处理，而不是让整个对话崩溃。
- 流式 Agent 的工具结果摘要现在会把工具执行错误以脱敏短句展示给前端，不再把非 JSON 错误误显示为“工具调用已完成”。
- 流式 Agent 也会识别工具返回的结构化 JSON 错误，并按错误态摘要展示，避免“查无消息”等工具失败在前端显示成绿色成功。
- 空 embedding 输入不会初始化外部客户端，非法 embedding batch size 会回退到安全默认值。

### 聊天会话与前端对话体验

- 聊天请求会 trim `question` 和 `session_id`，空白会话 ID 会被拒绝；前端发送、本地展示和停止生成保存都统一使用 trim 后的问题文本。
- 会话 path 参数和批量删除的 `session_id` 增加长度限制，异常客户端不能把超长 ID 压到数据库查询层。
- 停止生成接口会校验并 trim question，拒绝纯空白问题；部分回答会在前后端统一 trim 并限制到 20000 字符，避免长回答停止请求校验失败；客户端断连后的部分回答只保存一次，避免重复写入停止记录。
- 用户停止生成且部分回答已保存后，若底层模型流随后只返回迟到错误或抛异常，后端会按“已停止”正常收尾，不再把该会话误标为错误或重复写入回答。
- 聊天 SSE 错误事件在路由出口再次脱敏后再发给前端；无法解析的错误 payload 会替换为通用错误，避免未来流式生产者绕过下游脱敏时泄露敏感信息。
- 前端停止生成请求改用停止流程自己的短超时，页面卸载清理时也复用该时限，避免慢网络下停止请求按普通接口超时长时间悬挂。
- 会话列表和消息分页增加安全参数解析，内部脚本或未来扩展传入异常分页值时不会直接抛 `ValueError`。
- 后端聊天会话时间戳提升到毫秒级，连续生成、重命名或删除后的会话列表排序更贴近真实最近操作顺序。
- 会话批量删除内部改为分批查询和删除，即使未来脚本或前端提高批量上限，也不会因为一次性 `IN (...)` 参数过多撞到 SQLite 限制。
- 前端聊天输入限制到后端一致的 8000 字上限；生成中或禁用状态变化时会关闭建议弹层。
- 聊天页面卸载时会让晚到的 SSE 回调失效，避免切页或刷新后旧流继续修改页面状态或弹出迟到提示。
- 会话列表的完整刷新现在会取消并使旧分页请求失效，避免“加载更多”的迟到响应在删除、重命名或生成完成刷新后把旧会话重新合并回侧栏。
- 会话列表完整刷新期间会阻止新的“加载更多”请求，避免慢网络或快速点击时用旧 offset 合并分页结果。
- 生成完成、停止生成、重命名、单个删除和批量删除后的会话侧栏刷新会强制中止旧列表请求并重新拉取，避免慢网络下旧请求把操作前的会话状态写回侧栏。
- 后端重命名接口也会拒绝运行中或停止中的会话，直接 API 调用会返回 `409`，避免绕过前端保护修改正在生成的会话元数据。
- 消息列表加载更早历史时，如果请求失败或返回页没有新增消息，会清理临时滚动锚点，避免下一次发送消息时滚动位置被错误恢复到旧位置。
- 连续提问或停止生成并成功保存后，前端会同步已加载消息数和总消息数，避免随后加载更早历史时用旧 offset 拉回已显示的新消息。
- 切换会话并加载历史消息期间会暂时禁用发送入口，避免慢网络下用户抢先发送后又被迟到的历史消息响应覆盖本地新消息和流式状态。
- 会话生成中会退出侧栏重命名编辑，并禁止对运行中/停止中的会话发起重命名，避免输入框失焦绕过禁用态提交会话元数据变更。
- 会话侧栏支持更稳健的批量选择、删除、重命名、忙碌状态过滤、非法日期隐藏和状态颜色 token。
- 工具调用结果在前端会识别 `工具执行错误` 等失败摘要并使用错误态展示，不再把失败工具结果显示成绿色成功。

### 设置、健康诊断、日志与安全

- 新增统一脱敏模块，覆盖后端日志、错误响应、导入进度消息、工具参数预览、异常文本和前端可见 action，避免 API Key、token、secret 等敏感内容进入日志或 toast。
- 脱敏规则补强 URL 用户信息、`password/passwd` 字段和 URL query 参数里的密钥值；保留非敏感 query 参数，避免排障日志既泄露密码又丢失必要上下文。
- 脱敏规则继续补强带引号的敏感字段值，`"api_key": "..."`、`password="..."` 等值里即使包含空格或转义引号也会整体遮蔽，避免日志残留半截密钥。
- 前端 Markdown 渲染收紧允许标签，只保留文本结构、代码块、表格和链接，禁止图片、iframe、音视频等自动加载外部资源的标签，避免模型输出触发隐私外联请求。
- 前端 Markdown 链接增加协议白名单，并为外部 HTTP/HTTPS 链接补充 `target="_blank"` 与 `rel="noopener noreferrer"`，避免模型生成链接导致页面跳走或反向标签页风险。
- 后端错误响应在公共 `error_payload` 层统一脱敏 `message`、`action` 和 `details`；HTTP 异常处理也复用同一规则。
- 日志系统输出 JSONL 诊断日志，支持近期日志 API、日志级别校验、损坏行跳过、用户可见消息翻译和敏感信息脱敏。
- 近期日志接口改为从日志文件尾部倒序读取，找到足够的匹配记录后即停止，避免日志文件变大后前端日志面板刷新越来越慢。
- 近期日志接口在当前日志不足时会继续读取 `.1` 轮转日志，避免刚发生轮转后前端排障面板看不到轮转前的最近错误。
- 日志面板已有结果时刷新会保留当前列表，只在刷新按钮中显示加载状态；接口失败后仍展示上次成功结果和持续错误提示，避免排障时列表短暂消失。
- 健康诊断更细分数据库、聊天模型、embedding 配置、向量索引和会话分块状态；能区分空库、有消息但无分块、缺向量、缺 embedding 配置等场景，并给出对应行动建议。
- 健康接口补充 `summary_model_configured`、`summary_model`、`summary_model_missing` 和 `vector_index_available`，前端可据此隐藏当前环境无法执行的摘要或向量构建模式。
- 设置 API 支持持久化非密钥运行时配置到 `runtime/backend_settings.json`，API Key 仍只从环境变量读取；保存与环境变量相同的模型/超时不会冻结为旧 override。
- 设置 API 会 trim 并限制单个启用工具名长度，直接 API 调用也不能用超长工具名污染校验错误、日志或运行时状态。
- 设置响应新增只读 `available_tools`，前端根据后端注册表渲染工具开关，避免工具新增或下线时继续依赖硬编码列表；该字段不会写入持久化设置文件。
- `chat_model: null` 和 `chat_timeout: null` 现在表示清除运行时 override 并回到环境变量；加载持久化设置时也遵守这个语义。
- 对话和 embedding SDK 客户端缓存会把 endpoint、API Key、timeout 和 retry 配置纳入缓存键，避免运行中配置变更后健康检查已看到新配置但实际请求仍复用旧客户端。
- 设置保存增加原子写入、失败回滚、非法工具拒绝、空 system prompt 拒绝、数值范围校验和重置失败保护。
- 设置页加载增加请求序号保护，连续重试或慢网络下旧请求不会覆盖新请求的加载状态或错误提示。
- 设置保存或重置请求进行中会锁定整组表单控件，避免用户继续编辑后又被后端响应回写覆盖。
- `.env.example` 中的 `example.com`、`sk-...`、`your-*` 等占位值会被视为未配置，避免误连模板地址。
- 模型端点、API Key 和模型名在健康检查与 SDK 初始化前都会去掉首尾空白，避免手工编辑 `.env` 后出现“显示已配置但实际请求失败”的错位。
- 默认 CORS 覆盖本地 Vite 常用备用端口，前端端口从 5173 漂移到 5180 范围内仍可访问后端。

### 统计、建议与辅助接口

- 统计接口补充分页线程/发送人列表、稳定排序、总数返回和空白名称归并，方便前端展示大盘数据。
- 建议接口支持发送人和会话合并补全，正确转义 LIKE 通配符；空查询时返回热门实体。
- 建议 HTTP 接口与 WebSocket 一样会对超大查询直接返回空建议，避免异常客户端用无意义长输入触发数据库查询。
- 建议 WebSocket 增加消息大小限制，超大输入直接返回空建议，不再退化为热门实体查询，避免异常输入触发不必要的数据库访问。
- 日志、统计、建议和健康面板前端统一处理非法日期、空状态、加载状态、错误提示和响应式布局。
- 日志面板加载失败时会显示持续可见的错误提示，并保留上次成功加载的日志，避免把接口失败误展示成“暂无日志”。
- 统计面板初次加载全部失败时只显示错误和重试入口，不再展示默认 0 大盘；已有成功结果或部分结果时仍保留当前数据并显示错误提示。
- 统计表切换每页条数后如果刷新失败，会恢复到上一次成功加载的条数，避免旧表格数据和新分页规则混合显示。

### 前端工程与界面完善

- 前端 API 层补齐导入、任务、日志、统计、建议、会话批量删除、设置保存等接口封装，并对 path 参数做 URL 编码。
- 设置页拆分为对话设置、数据导入、健康诊断、运行日志和统计概览等实际工具面板；未知路由会回到聊天页。
- 导入面板新增文件列表刷新、上传队列、任务行合并、批量进度、任务实时连接、日志尾部、模式限制和禁用态展示。
- 上传 JSON 会写入稳定聊天 scope 元数据，解析器优先用该 scope 生成消息 ID 前缀；scope 优先使用 WeFlow 会话稳定 ID，缺失时退回会话显示名，使同一聊天记录重复上传、改名上传或追加导出时能够增量合并，不再因为上传文件 UUID 或追加导出起点变化而重复入库。追加导出只包含回复消息时，入库层也会用稳定 scope 归一化 `reply_to`，避免上下文追溯断链。
- 导入面板的文件列表、任务列表和索引诊断加载增加请求序号保护；任务刚启动或上传后遇到慢网络时，旧响应不会覆盖新的文件状态、任务进度或诊断结果。任务启动成功后，对应文件行会立即显示为导入中并关联新任务，不必等待下一次文件列表刷新。
- 导入任务启动请求等待后端响应期间，单文件和批量启动入口会全局禁用，避免慢网络下连续点击让多个启动请求竞态并依赖后端 409 兜底。
- 导入面板把上传中、批量中、启动请求中和已有运行任务统一收敛为启动锁，减少模板和点击处理里的禁用态漏判。
- 导入任务进度展示会在任务已经进入完成、失败或取消终态时忽略旧 WebSocket 运行中进度，避免状态徽标已经终态但进度条仍显示处理中。
- 设置页支持用 `?tab=ingest` 直接打开数据导入面板；即使后端暂不可连接，上传入口也会保留在页面上，并在文件/任务区域显示可重试错误态。
- 聊天 Markdown 标题样式按消息气泡场景收紧，不再使用浏览器默认页面级标题尺寸，避免模型输出长标题时挤压或遮挡对话内容。
- 统计面板、日志面板、健康面板、会话侧栏、聊天消息和设置面板提取重复样式到 scoped CSS，减少内联样式和重复空状态样式。
- Toast 增加复杂错误对象规范化和活跃重复消息去重，避免相同错误反复刷屏。
- Toast 现在会正确提取 FastAPI/Pydantic 风格 `detail` 数组或 `error.details` 中的错误消息，避免真实校验错误显示成 `[object Object]`。
- 前端 API 错误解析会提取嵌套 `error.details`、`detail` 数组和字段路径，上传、导入、设置、日志等接口失败时能给出更接近真实原因的提示。
- 前端会话改名/删除/批量删除、设置保存/重置以及导入任务启动/取消请求现在支持外部取消信号；页面卸载时会终止仍在等待的普通操作请求，避免慢网络下旧组件继续等待或弹出迟到提示。
- 聊天 SSE 请求增加连接阶段超时提示；响应建立后会清除连接超时，避免长回答被普通请求超时误中断，同时仍保留用户手动停止生成。
- 健康面板在空库时会把向量搜索状态显示为“暂无数据”，避免把尚未导入聊天记录误导成向量配置故障。
- 健康面板刷新改为只请求一次详细诊断，并在前端派生顶部摘要，避免 `/api/health` 和 `/api/health/diagnostics` 重复执行同一组数据库与向量索引检查。
- 健康面板已有诊断数据时刷新会保留当前结果，并在刷新失败后显示持续可见的错误条和重试入口，避免排障时页面清空或只靠短暂 toast 提示。
- 健康诊断现在返回可点击的 `action_target`，前端只对能在应用内处理的建议显示跳转按钮，避免把缺 API Key、Base URL 或 `EMBED_*` 的环境配置误导到设置页。
- 健康诊断和后端启动日志改用轻量统计路径，不再为刷新状态额外计算完整消息类型分布，并合并核心计数聚合，降低大库环境下设置页刷新成本。
- 统计面板刷新会保留当前会话/发送人分页位置，不再把大库排查中的表格重置回第一页；初次进入和手动切换每页条数仍从第一页加载。
- 前端构建依赖补齐，`package.json` 和 `package-lock.json` 保持运行时依赖一致。
