# 更新日志

## 2026-06-19

### 文件级导入范围与旧库兼容

- 修复导入面板中文件状态的业务口径：后端新增并使用 `JSON 文件 -> 实际消息 ID -> 会话块` 的来源映射，已导入旧库不再把单个 JSON 显示成 `会话块 0`；单文件的仅摘要、仅向量、仅 FTS、仅分块等操作也会按目标 JSON 收窄范围。
- 文件列表接口不再为了展示旧数据状态隐式解析大 JSON；缺少来源映射的旧导入文件会返回 `null` 索引明细，等用户明确执行全流程导入或重建后再补齐映射。
- 未导入文件的索引计数不再显示为 `0/0/0`，避免用户误以为索引已检查通过。
- 文件列表刷新改为批量读取来源映射、关联会话块和索引缺口统计，减少已导入 JSON 较多时的重复数据库查询。
- 单个已同步 JSON 执行增量、全流程、仅分块、仅摘要、仅向量时，会按该 JSON 关联消息、线程和会话块限定范围；FTS 仍保持全量重建语义以保证 external-content FTS5 正确性。
- 单个 JSON 使用强制重建时，会优先按目标 JSON 关联线程范围重建分块、摘要和向量，避免用户处理一个文件却牵动无关数据。
- 目录目标的单项构建增加预检：空目录、无 JSON 目录、从未入库的目录、没有可定位消息范围的目录会提前失败；仅摘要和仅向量还要求已有会话分块。
- CLI 的 `--skip-import` 单项索引构建也要求目标路径至少匹配已入库 JSON，避免 `--force-fts`、`--force-summary` 等命令对未导入文件静默完成 0 条修复。
- 显式单项模式严格只执行用户选择的阶段：仅 FTS、仅分块不会因为自愈逻辑额外调用摘要或 embedding；仅向量不会为了缺摘要而调用摘要模型。普通增量、全流程和强制重建仍保留自动补齐逻辑。
- 全流程导入语义修正：`incremental` 仍按文件变化跳过，`full` 通过 `--force-import` 强制重新解析目标 JSON，`rebuild` 通过 `--force-rebuild` 强制重建相关索引。
- 解析规则升级后，旧 JSON 即使大小和 mtime 未变也会被识别为“需重新导入”；`ingest_files` 增加 `parser_version`，旧版本记录会被视为过期。
- JSON 成功解析入库后，即使后续摘要或 embedding 因模型/API 错误停止，也会先保存该文件的导入记录，让前端能继续显示已入库状态和缺摘要/缺向量数量。
- 单 JSON 摘要或向量修复按目标会话 ID 分批查询已有向量，不再扫描全库向量表。
- 文件来源映射只记录数据库中真实存在的消息 ID，避免旧脚本或异常重导留下悬空映射后误导文件级状态。

### WeFlow 解析规则与消息 ID 稳定性

- 解析器扩展真实微信语义消息收录：系统消息、聊天记录、链接、转账、位置、小程序、文件名、通话记录、名片、动画表情说明、其他带文本载荷的消息会进入索引；纯图片、视频、无转写语音等仍跳过。
- 裸 `[表情包]`、`[引用 [消息]]` 等无语义占位符会被丢弃；带 `emojiCaption` 或真实引用正文的表情仍保留。
- 顶层消息列表兼容 `messageList`、`message_list`、`msgList`、`chatRecords`、`records`，Web 上传校验和 CLI ingest 都接受这些真实导出结构。
- 消息类型兼容数字 `type/msgType/messageType` 与 `text/image/voice/video/location/link/file/system` 等短类型别名。
- 解析规则兼容 `typeName/msgTypeName`、`formattedTime/msgCreateTime`、`msgContent/plainText`、`newMsgId/clientMsgId/localMsgId`、`quote/refer/reply` 嵌套引用对象等导出形态。
- 字段别名解析会跳过空字符串、`null` 和无效时间主字段，继续读取后续有效别名，避免占位字段导致消息被误跳过。
- 嵌套引用对象会跳过空的 `quote/refer/reply` 占位对象，并从同一个有效引用对象读取发送人、正文和被引用消息 ID。
- 所有导出消息 ID 都改为带文件作用域：`platformMessageId/serverId`、`msgId/messageId/id`、`localId` 和 fallback index 都不会在多文件导入时互相覆盖。
- 文件作用域从仅文件名升级为项目内相对路径；项目外文件使用父目录哈希加文件名，避免把用户机器绝对路径暴露进 message_id。
- 上传 JSON 会写入稳定聊天 scope；解析器优先用该 scope 生成消息 ID 前缀，使同一聊天重复上传、改名上传、追加导出可以增量合并。
- 稳定 scope 优先使用 WeFlow 会话稳定 ID，缺失时退回会话显示名，解决追加导出起点不同导致 scope 变化的问题。
- 旧库兼容升级：早期裸平台 ID、raw ID、basename scoped ID、随机上传 UUID scoped ID 都会在重导时尽量原地升级为新 scoped ID，避免重复入库。
- `reply_to` 入库时会从 raw ID、basename scoped ID、稳定 upload scope 等候选中规范到同文件作用域下已存在的消息 ID，并避免坏数据生成自引用。
- 长路径 scoped `message_id` 不再在工具层或 `get_context` 中被 200/240 字符截断，适配多账号目录、同名 JSON 和长平台消息 ID。
- 多行聊天消息写入会话块时，每一行都会重复发送人前缀，避免摘要或 embedding 阶段把续行误判成无人称文本。
- 跨日但连续的聊天分块标题会保留结束日期，避免检索证据显示成同一天的时间范围。
- `--force-rebuild` 会真正重新解析未变化 JSON，用于解析器升级后把新支持的消息类型补进库。

### Ingest 前端与任务体验

- 导入页在真实旧库中验证并修正了索引诊断展示：可显示 `0/2789` 向量索引缺口，并把健康诊断直接接到数据导入页顶部。
- 文件状态文案从“文件已变化”改为“需重新导入”，覆盖文件内容变化和解析规则升级两类场景。
- `changed / 需重新导入` 文件默认选择全流程导入；未导入文件只显示全流程和强制重建；已同步文件才开放单项索引模式。
- 模式下拉和开始按钮补充 hover 提示，明确增量、全流程、仅 FTS、仅分块、仅摘要、仅向量可能做什么以及是否调用模型/API。
- 后端 `/api/ingest/start` 返回按模式定制的启动消息，前端 toast 优先显示后端消息。
- 已完成、已失败或已取消的任务会忽略旧 WebSocket 运行中进度，避免徽标已终态但进度条仍像在处理。
- 已完成、已取消或失败的导入任务不再根据旧进度事件显示 ETA。
- 导入任务列表和 WebSocket 进度只读取最近日志片段来推断旧式文本进度，避免大日志拖慢任务面板刷新。
- 任务时间戳提升到毫秒级，同一 JSON 快速重试时用稳定排序选择最近任务，避免文件行展示旧任务。
- 任务列表完整刷新期间阻止新的“加载更多任务”请求，避免慢网络或快速点击时用旧 offset 合并分页结果。
- 静默刷新会按后端分页上限补齐已加载页，并只继承同 ID 任务的本地进度字段，避免旧任务列表被压回第一页或继续展示已移除任务。
- 文件列表已有数据时，刷新失败会保留当前列表并显示持续错误提示，不再把后端不可达误展示成“暂无可导入文件”。
- 默认前端 API 地址改为按当前页面主机连接 `:8000`，减少 Windows 上 `localhost` 与 `127.0.0.1` 混用导致的空列表问题。
- 同名 JSON 文件卡片显示相对路径，并把完整 `file_id` 放入 hover title，避免 `data/` 和 `imports/` 下同名文件被点错。
- 上传、批量导入、启动请求、运行中文件行、取消中状态统一纳入启动锁，避免慢网络下连续点击或任务列表滞后导致重复启动。
- 文件列表、任务列表、健康诊断、批量轮询、取消任务、WebSocket 回调和上传/批量 finally 都增加组件卸载保护，避免切页后写状态或弹迟到 toast。
- `getIngestFiles()`、`getAllIngestFiles()` 支持 `AbortSignal`；页面卸载时会主动取消仍在等待的文件列表分页请求。
- 上传 JSON 校验通过但 `.meta` 元数据保存失败时，会删除刚写入的 JSON 和残留 meta，并返回明确 500，避免半成功文件出现在列表。
- 上传校验按探测到的 JSON 编码从临时文件文本流解析，兼容 UTF-8 BOM / UTF-16，同时避免为大 JSON 额外复制整份 bytes。

### 检索、Agent 与工具策略

- FTS 消息搜索和会话搜索会把双引号、冒号、括号、加号、连字符等用户输入当作普通文本处理，不被 FTS 查询语法误伤。
- 关键词检索命中长消息时，预览会围绕第一个命中词截取，避免结果显示“命中了但看不到命中内容”。
- 语义检索带群聊或时间过滤时，会把过滤条件下推到 `sqlite-vec` 候选范围内，避免目标群聊的相关会话块被全库 top-k 挤掉。
- 语义检索最终线程过滤与 SQLite `LIKE` 一样大小写不敏感，避免英文群名大小写不同导致命中被误过滤。
- 语义检索只在过滤条件规范化后确实有效时才启用“过滤范围内最近会话”兜底；空白会话名或无效日期不会误触发最近会话兜底。
- 语义检索存储层会裁剪超长 query，避免 direct-call 绕过工具 schema 后把无界输入发送给 embedding 服务。
- Agent 每轮会把当前启用/停用工具策略追加到系统提示词；即使自定义提示词保留旧工具路由，模型也不会继续被引导调用已停用工具。
- 流式工具结果摘要能识别结构化 JSON 错误和工具执行错误，并以错误态展示，不再把失败工具结果显示成成功。

### 设置、健康、日志与统计面板

- 健康诊断返回可点击的 `action_target`；前端只对能在应用内处理的建议显示跳转按钮，缺 API Key、Base URL 或 `EMBED_*` 时只显示文字建议。
- 健康面板在空库时把向量搜索状态显示为“暂无数据”，避免把尚未导入误导成向量配置故障。
- 健康面板刷新只请求一次详细诊断，并在前端派生顶部摘要，减少重复数据库和向量检查。
- 健康面板、日志面板、统计面板、设置面板、导入面板在组件卸载后都会忽略迟到异步结果，避免写已卸载组件状态或弹迟到提示。
- 设置页加载、保存、恢复默认请求增加卸载保护；保存或重置请求进行中会锁定表单控件，避免用户继续编辑后被响应覆盖。
- 设置页连续重试或慢网络下旧请求不会覆盖新请求的加载状态或错误提示。
- 日志接口从日志文件尾部倒序读取，当前日志不足时继续读取 `.1` 轮转日志；日志面板刷新失败时保留上次成功结果并显示持续错误。
- 统计面板刷新会保留当前会话/发送人分页位置，不再跳回第一页；切换每页条数后如果刷新失败，会恢复到上一次成功条数。
- 建议 HTTP 接口与 WebSocket 一样对超大查询直接返回空建议，避免异常客户端用无意义长输入触发数据库查询。
- 脱敏规则补强 URL 用户信息、`password/passwd` 字段、URL query 参数密钥值和带引号敏感字段值，减少日志残留半截密钥的风险。
- 前端 Markdown 渲染收紧允许标签和链接协议，禁止图片、iframe、音视频等自动外联资源；HTTP/HTTPS 外链会补 `target="_blank"` 与 `rel="noopener noreferrer"`。
- Toast 能正确提取 FastAPI/Pydantic 风格 `detail` 数组、`error.details` 和字段路径，避免校验错误显示成 `[object Object]`。

### Web 导入功能闭环

- 设置页新增并完成 JSON 聊天文件导入工作台，支持多 JSON 上传、逐文件 ingest 状态、全流程导入、增量导入、强制重建、仅 FTS、仅分块、仅摘要、仅向量、批量导入、取消任务、任务列表、进度条、阶段文案、ETA、日志尾部、WebSocket 实时进度和轮询兜底。
- 后端新增安全上传、WeFlow JSON 校验、任务状态分页、单任务串行、取消子进程、任务裁剪、日志脱敏、进度事件和单项索引模式限制。
- 导入子进程新增结构化进度事件，普通 CLI 保持文本输出；后端只把结构化事件用于任务状态，不把内部事件混入用户可见日志。
- 后端关闭时会取消运行中和排队的导入任务并更新状态，避免重启后留下误导性运行状态。
- 导入失败会优先从结构化失败进度或最近日志尾部提炼脱敏错误原因，避免前端只显示退出码。
- 上传接口拒绝空文件、损坏 JSON、非 WeFlow JSON 和路径穿越；上传内容先写入不可导入临时文件，校验成功后再原子替换为最终 `.json`。
- 损坏或非 UTF-8 JSON 上传会返回 400 并清理临时文件，不再变成内部 500。
- 文件级运行态会禁用单文件和批量启动入口，避免任务列表短暂滞后时误点。
- 后台任务启动成功后，前端会立刻插入临时任务行并连接 WebSocket，再静默刷新列表，减少慢 GET 带来的“看不到任务”体验。
- 导入面板和日志面板会隐藏非法时间，不再显示 `Invalid Date`。
- 前端文件列表同名文件初步支持展示区分信息；移动端统计页、会话侧栏触控目标和多处空状态/加载状态也做了适配。

### 摄入流水线与解析兼容

- `core.ingest` 支持递归扫描目录内 JSON，并按稳定路径顺序处理；单个 WeFlow 文件解析失败时可继续处理其他文件，全部不可用时返回清晰错误。
- 新增 `--skip-import` 和单项索引模式，用于只基于已有数据库补建 FTS、分块、摘要或向量，避免用户选择单项构建时误导入尚未入库 JSON。
- 摘要构建可以只依赖显式 `SUMMARY_MODEL`，不再错误要求默认 `CHAT_MODEL` 存在，适合先建库、后补摘要的分阶段配置。
- 摘要和 embedding 失败信息会脱敏并压缩，配置缺失时给出可执行提示；向量维度仍可按模型实际返回自动修正。
- WeFlow 解析器初步兼容更多常见字段别名，包括 `msgType/messageType/msg_type`、`timestamp/createdAt/create_time`、`text/message/body`、`messageId/local_id/reply_msg_id`、引用正文和引用发送人等。
- 分块逻辑在切分前稳定排序消息，尾部小块会在时间距离允许时合并回前一块，减少过碎会话块影响摘要和检索效果。

### 检索、Agent 与聊天会话稳定性

- 工具参数窗口限制前移到 Pydantic schema：关键词检索、语义检索、上下文追溯和时间浏览会在进入数据库前拒绝过大的请求。
- 检索存储层增加 direct-call 兜底：非法 `limit`、`offset`、`before`、`after`、时间过滤和空查询不会直接抛异常，而是回退、夹紧或返回结构化空结果。
- 语义检索在没有 query、没有向量、旧库 JSON 字段损坏、过滤条件异常等情况下保持可预期返回。
- 关键词检索对短词使用 `LIKE` 兜底，并限制过多查询词。
- Agent 直接调用会先规范化问题，空问题或非字符串问题不会初始化模型；历史裁剪会移除过期工具消息和非法开头上下文。
- 普通和流式 Agent 都能处理畸形 tool call：错误会转成可读工具错误继续处理，而不是让整个对话崩溃。
- 空 embedding 输入不会初始化外部客户端，非法 embedding batch size 会回退到安全默认值。
- 聊天请求会 trim `question` 和 `session_id`，空白会话 ID 会被拒绝；前端发送、本地展示和停止生成保存都统一使用 trim 后的问题文本。
- 会话 path 参数和批量删除的 `session_id` 增加长度限制；批量删除、停止生成、断连部分回答保存等路径补充重复写入和空白输入保护。
- 前端聊天输入限制到后端一致的 8000 字上限；生成中或禁用状态变化时关闭建议弹层。
- 聊天页面卸载时让晚到的 SSE 回调失效，避免切页或刷新后旧流继续改页面状态。
- 会话列表和消息分页增加安全参数解析，内部脚本或未来扩展传入异常分页值时不会直接抛 `ValueError`。

### 设置、健康、日志、安全与辅助接口

- 新增统一脱敏模块，覆盖后端日志、错误响应、导入进度消息、工具参数预览、异常文本和前端可见 action，避免 API Key、token、secret 等敏感内容进入日志或 toast。
- 后端错误响应在公共 `error_payload` 层统一脱敏 `message`、`action` 和 `details`；HTTP 异常处理复用同一规则。

## 2026-06-13

### 摄入性能优化

- 优化 `python -m core.ingest data` 的增量摄入流程：冷启动全量导入从 485.1s 降至 6.4s（76x），重新导出后的增量导入从 451.4s 降至 9.5s（47x），向已有库追加新聊天从 81.7s 降至 6.7s（12x）。
- 将 `recompute_message_sequence` 从全量关联子查询改为基于主键联接的 `UPDATE ... FROM`，并限定受影响线程、跳过未变化记录，12.3 万条消息的 seq 重算从约 477s 降至约 1s。
- 增量导入只重建有新消息的线程；`sessions` 新增 `text_hash`，内容未变化的会话块会复用既有摘要和向量，避免全库重新分块、重新摘要、重新 embedding。
- 新增自愈逻辑：每次运行自动补齐缺失的 FTS 索引、seq、摘要和向量；中断或部分失败后再次运行即可续传。
- 摘要和 embedding 改为两阶段流水线并发执行，摘要完成后立即进入 embedding 批队列；embedding 默认并发为 4，摘要写库改为批量提交。
- 复用 LLM/embedding 客户端，减少连接建立开销；对 bge-m3 关闭 OpenAI `tiktoken` 本地分词，直接发送原文。
- 修复 external-content FTS5 表的 `sync_missing_fts` 判断错误，避免普通导入后全文索引为空。
- embedding 单批失败不再中止整个导入，失败项会在后续自愈运行中补齐；摘要失败会按错误类型统计输出。

### 基准与验证

- 新增 `bench/` 基准工具、mock LLM 服务、基线代码快照和数据库一致性校验脚本，用于复现优化前后对比。
- S2 增量导入的 LLM 调用从 2,789 次摘要 + 88 批 embedding 降至 128 次摘要 + 4 批 embedding。
